## Context

SnapCTRL is a desktop application that provides a native GUI for controlling Snapcast multi-room audio systems.

## External Actors

| Actor | Role | Interface |
|-------|------|-----------|
| **User** | Controls audio, views status | Qt GUI |
| **Snapcast Server** | Streams audio, manages clients | JSON-RPC over WebSocket |

## Assumptions

1. Snapcast server is running on local network
2. Server uses default port 1704
3. No authentication required (Snapcast has none)
4. Single server per user (MVP)

## Constraints

| Constraint | Impact |
|------------|--------|
| Cross-platform (Win/macOS/Linux) | Qt framework required |
| Real-time updates | WebSocket required |
| Native look & feel | Platform-specific theming |
| <200MB memory | Efficient state management |
